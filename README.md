# LoyaltyLite

LoyaltyLite is a Flutter app for small businesses to manage customers and loyalty points with Firebase backend services.

## Objective Coverage

This project demonstrates:

- Firebase Authentication for secure user sign-in (phone OTP flow).
- Cloud Firestore for real-time customer and loyalty data sync.
- Firebase initialization and platform setup using FlutterFire.
- Optional Firebase Storage integration path for media uploads.

## 1) Firebase Setup for Flutter

### Create and connect Firebase project

1. Open Firebase Console and create a new project.
2. Add your Android app (`com.smartkirana.app`) and/or iOS app.
3. Download and place platform config files:
   - Android: `android/app/google-services.json`
   - iOS: `ios/Runner/GoogleService-Info.plist`
4. Configure FlutterFire from project root:

```bash
flutterfire configure
```

5. Ensure dependencies are installed:

```bash
flutter pub get
```

### Required dependencies

Already present in this project:

```yaml
dependencies:
	flutter:
		sdk: flutter
	firebase_core: ^3.0.0
	cloud_firestore: ^5.0.0
	firebase_auth: ^5.0.0
```

### Firebase initialization

Firebase is initialized in `lib/main.dart`:

```dart
Future<void> main() async {
	WidgetsFlutterBinding.ensureInitialized();
	await Firebase.initializeApp(
		options: DefaultFirebaseOptions.currentPlatform,
	);
	runApp(const LoyaltyLiteApp());
}
```

## 2) Key Firebase Services Used

| Service                     | Purpose                      | In this app                                     |
| --------------------------- | ---------------------------- | ----------------------------------------------- |
| Firebase Authentication     | User sign-up/sign-in/session | OTP-based phone login for business owner        |
| Cloud Firestore             | Real-time NoSQL data store   | Businesses, customers, and loyalty transactions |
| Firebase Storage (Optional) | Store media files            | Can be added for profile images/docs            |
| Cloud Functions (Optional)  | Serverless backend logic     | Can be added for notifications/automations      |

## 3) Authentication Implementation

- Auth logic is handled in `lib/services/auth_service.dart`.
- App state management for login flow is in `lib/providers/auth_provider.dart`.
- UI is in `lib/screens/login_screen.dart`.

Current flow:

1. User enters phone with country code.
2. App sends OTP via Firebase Auth.
3. User verifies OTP.
4. Auth state stream updates and app proceeds to setup/dashboard.

UX feedback:

- Error messages are shown when OTP send/verify fails.
- Success SnackBars are shown when OTP is sent and login succeeds.

## 4) Firestore Real-Time Sync

Realtime sync is implemented with Firestore streams:

- `streamBusinessByOwner(...)`
- `streamCustomers(...)`

These streams are consumed by providers (`BusinessProvider`, `CustomerProvider`) and push updates to UI through `notifyListeners()`.

Why this is real-time:

- Firestore `snapshots()` keeps a live subscription.
- When one user/device writes new customer or transaction data, all connected clients with the same query receive updates automatically.
- No manual refresh is required in the dashboard customer list.

## 5) Optional Firebase Storage Integration

If your app needs media uploads, add:

```yaml
dependencies:
	firebase_storage: ^12.0.0
```

Then run:

```bash
flutter pub get
```

Example upload method:

```dart
import 'dart:io';
import 'package:firebase_storage/firebase_storage.dart';

Future<void> uploadFile(File file) async {
	final ref = FirebaseStorage.instance.ref().child('uploads/myImage.jpg');
	await ref.putFile(file);
}
```
